Iteration Log
How the app evolved
Each round of feedback turned into a concrete set of changes with no up-front spec, just continuous refinement.
1
Sprint 1: Core build
Shipped the first working app
Started from a brief: "I want a Mac desktop app that tracks my daily water intake and reminds me to drink." Built and packaged as a .dmg in one session. No prior wireframes, built straight from conversation.
3L daily goal (SGT reset) Water / Milo / Yakult quick buttons Animated glass visual Today's log with timestamps Packaged as .dmg Smart reminders (90 min inactivity)
2
Sprint 2: Log improvements
Liquid types + inline editing
Realised after using it that the log needed to track what was drunk, not just how much. Also needed a way to fix mistakes without undoing everything.
Drink type stored per entry Type dropdown on custom input Inline edit without leaving the screen Backwards compat for old entries
3
Sprint 3: History page
Calendar, stats & day detail
Wanted to look back and understand patterns. Designed and built a full history page with a calendar heatmap and summary statistics.
Calendar heatmap (hit / partial / miss) Date range pills (7 / 14 / 30 / all) Daily avg, best day, days on target Month navigation + dropdown Click-to-expand day detail panel
4
Sprint 4: Custom drinks (design)
Beyond Water, Milo & Yakult
Wanted to add personal drinks (Hojicha, barley water, etc.) without being locked to three hardcoded types. Iterated through several mockup rounds before settling on the right UX.
Add / edit / delete custom drink types 8-colour palette per drink Colour-coded type labels in log Manage drinks page (gear icon) Notes on individual drink entries
5
Sprint 5: Layout & UX
Scroll, free-text & better glass
Feedback on the mockup led to three UX improvements: the button grid became a horizontal scroll (less visual weight), the custom input became free-text so any drink can be logged on the fly, and the glass labels were moved outside for readability.
Horizontal scroll drink row Free-text drink type on custom input Glass labels outside with connector lines Notes shown in history day detail too
What it does
Full feature set
Everything that ended up in the product across all iterations.
💧
Daily tracking with 3L goal
Auto-resets at midnight SGT. Animated glass fills as you log. Progress message changes with context.
⚡
Quick-log buttons
Single tap to log a favourite drink at its default amount. Scrollable row, so add as many as you want.
✏️
Custom drinks + free log
Add named drink types with a custom colour. Or just type any drink name in the freeform custom log row.
📝
Notes on entries
Attach a short note to any drink log ("after gym", "before meeting"). Shows in today's log and history.
🔔
Smart reminders
Native macOS notification if you haven't drunk for 90 minutes and are still below your daily goal. Only fires 8am–8pm SGT.
📅
History with calendar
Heatmap calendar showing hit/miss/partial days. Date range filter. Click any date for a full breakdown of what you drank.
📊
Stats for the period
Daily average, number of days on target, and best day, calculated over your selected date range.
🎨
Colour-coded log
Each drink type gets its own colour (blue for water, amber for Milo, etc.). Log entries are tinted without being overwhelming.
Under the hood
Tech stack
Deliberately kept simple, because the goal was a working Mac app, not a complex architecture.
🖥 Electron (macOS)
📄 HTML + CSS + Vanilla JS
🔒 Context isolation + preload bridge
💾 JSON file persistence (userData)
📦 electron-builder → .dmg
🛠 Node.js (no framework)
🍺 Homebrew + one-command setup script
