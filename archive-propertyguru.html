PropertyGuru Heuristic Evaluation

An academic usability evaluation of the PropertyGuru app using Nielsen's heuristics. Focused on three heuristics where we found the most meaningful gaps between user expectation and experience.

My roleUX researcher
MethodNielsen heuristic evaluation
ContextUniversity coursework
Heuristic 1
Visibility of system status

PropertyGuru showed inconsistent feedback during loading states. When a search took longer than expected, the app gave no indication of progress: no spinner, no skeleton state, no estimated wait. Users couldn't tell if the search was still running or had stalled.

Severity: High. For a property search app where queries can be slow (especially with filters applied), the lack of visible system status directly affects user trust and patience.

Recommendation: Add skeleton loading states for search results. Show a subtle progress indicator when filters are being applied. Use optimistic UI where possible to reduce perceived wait time.

Heuristic 2
Consistency and standards

The app used inconsistent interaction patterns across different listing types. Tapping a listing card on the search results page opened a full detail view, but tapping a similar card in the saved listings section opened a preview overlay. Users expected the same tap target to behave the same way regardless of where they were.

Severity: Medium. The inconsistency created mild confusion and a sense that the app hadn't been designed as a whole. Users had to relearn the navigation pattern in each section.

Recommendation: Standardise tap-to-open behaviour across all listing card types. If a preview overlay is used in one context, apply it consistently or replace it with the full detail view pattern.

Heuristic 3
Recognition over recall

The filter system required users to remember which filters they had applied and navigate to a separate filters page to modify them. Active filters weren't visually surfaced on the results page, so users who wanted to remove one filter had to re-open the full filters view.

Severity: High. Property searches are highly specific and iterative. Users typically apply several filters and expect to refine them easily. Forcing them to recall and re-navigate to their filter state creates friction in the core search loop.

Recommendation: Surface active filters as removable chips below the search bar on the results page. Allow one-tap removal without opening the full filters panel.
