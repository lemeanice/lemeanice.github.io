Noti Page Revamp

The notifications page had a layout problem that was quietly hurting both users and the business. Order updates were hard to find, non-order promotions were getting ignored, and the folder structure wasn't built to scale. We rethought the whole page.

Background
The notification page was doing too many things, badly

Shopee's notification page is where users go after they get a push notification. It's a high-traffic surface where people land daily to check order updates, see what promotions are active, and find anything else the app wants to tell them.

The problem was the structure. Notifications were grouped into folders (Orders, Promotions, etc.), but the design made it hard to tell at a glance which folder had new updates. Order status updates, which users care about most, weren't visually surfaced. And the folder tabs weren't built for the new types of content the team wanted to introduce.

Underneath the UX problem was a business problem: the page was underperforming as an engagement surface. Non-order promotional notifications were getting low click-through rates. A large portion of the addressable reach wasn't converting.

The problems we defined
Three things that needed to change
01
Order visibility was low
Users landing on the page couldn't immediately see their latest order status. They had to hunt for it inside a folder.
02
Promo reach wasn't being activated
The non-order promotional notifications had low AR (addressable reach) engagement. The layout wasn't giving them any visual priority.
03
The folder structure couldn't scale
We wanted to introduce new notification types, but the existing tab/folder design wasn't built to accommodate them cleanly.
Before
Folder-based layout. Order updates buried at the bottom.
After
Order updates pinned at the top. AI-powered Top Picks tab added.
What we built
Four changes to the notification experience
01 · Order visibility
Persistent order update card
The latest order status is now always visible at the top of the page with no folder needed. Users landing from any push notification can see their order at a glance.
02 · Personalisation ✦ AI-powered
Top Picks feed
A new AI-curated section that surfaces product recommendations personalised to each user. Top Picks pulls from Shopee's recommendation engine to show items the user is most likely to engage with, turning the notification page into a light discovery surface.
03 · Tab redesign
Restructured notification tabs
Redesigned the tab structure so notification types sit more logically. Cleaner visual separation between order updates, promotions, and other notification types.
04 · More AR components
Expanded promotional real estate
Added more addressable reach (AR) content components to the page layout, giving promotional notifications better surface area to convert.
Results
What the A/B tests showed

The test ran across five Southeast Asian markets. Results varied by market and treatment group; the range below reflects that spread.

+34% to +85%
Uplift in promo notification click-through rate. The range reflects variance across markets, with the strongest uplift in markets with lower baseline engagement.
+2.01%
GMV per user growth in key treatment groups, a meaningful business signal beyond just clicks.
Stable
Order notification engagement held steady. The redesign didn't come at the cost of the core use case.
Reflection
What I'd do differently

The biggest tension in this project was between order utility and promo engagement. These two goals pull in different directions: users want fast order updates, and the business wants promotional reach. We had to sequence the design so we didn't sacrifice one for the other.

If I were starting over, I'd have pushed to run a smaller test earlier. We spent a lot of time debating layout before we had any data. A lightweight A/B on just the order card position would have given us a faster signal and built more internal confidence before the full redesign.

The Top Picks feature was a bet I'm glad we made. It felt like a stretch at the time, adding a discovery surface to a utility page, but the data justified it. AI-powered personalisation on a high-traffic page like this has a surprisingly low bar to clear.
